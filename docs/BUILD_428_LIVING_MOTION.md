# BUILD 428 — Living Motion

> Vase 지시 2026-07-18 저녁. **기능 추가가 아니라 생명력 추가.** AI 변경 없음, 순수하게 움직임과 리듬.
> 한 줄 목표: **움직임을 추가하는 것이 아니라, 관찰하는 시간을 만든다.**
> 기준은 애니메이션 개수가 아니라 **'멈춤'의 질** — 꽃 앞에서 이유 없이 1초 더 머무는 순간.
> 원칙: 움직임은 귀엽기 위해서가 아니라 **관찰하기 위한 행동**이어야 한다.

## 0. 전제

- 캐릭터 그림체는 **soft v2로 통일** (Vase 판정 — `CHAR_STYLE_DEFAULT='soft'`). pixel·toon은 폴백으로 영구 보존.
- 조사 결과: **카메라 모션은 이미 있었다** — `startAction('record')`가 뜸(260ms)→플래시→셔터음, 렌더러가 카메라 포즈, 종료 시 AlbumCapture가 진짜 사진. 428은 그 앞에 '바라봄'을 붙여 스펙(멈춤→들기→찰칵→내림)을 완성한다.
- 하드룰 준수: 별이 상호작용은 props 기반만 — **가짜 대상을 지어내는 습관 금지**(하늘 보기·그냥 서 있기만 허용, 일기도 안 쓴다). 빼콩은 별이 경로를 바꾸지 않는다(427).

## 1-B. 428-B (배포됨) — 쉼·행동 다양화

- `actVariant` — 소품 종류 우선, 아니면 rng: **쉼** = 벤치·의자→앉기+발 흔들기 / 나무→기대기(1px 기움) / 그 외 앉기·쪼그리기(옷자락 바닥까지)·신발 보기(고개 푹). **사진** = 서서 / 무릎 꿇기 / 까치발. **관찰** = 서서 / 가까이 들여다보기(고개 1px)
- 자세는 **바라봄이 끝난 뒤에야** 갈린다 — 멈춤의 질 우선. 카메라 높이도 몸 높이를 따른다

## 1-C. 428-C (배포됨) — Snow Pace

> Vase 판정: 셋을 한꺼번에 넣지 않는다. **범위는 Snow Pace 하나.**
> "눈 오는 날은 별이가 덜 사는 게 아니라, 같은 하루를 더 천천히 걷는 것처럼 보여야 한다."

**구현** (`SNOW_PACE` 상수 — 하드코딩 금지 원칙):
- 별이 보행 애니메이션 템포 0.68배 · 발 딛는 간격 1.45배 · 상하 움직임 감소(sin>0.75일 때만 1px)
- 행동 종료 후 다시 걷기까지 0.5s 뜸 — **이 동안에도 worldX는 계속 흐른다**(진행률 계약)
- 빼콩은 0.82배 — 별이(0.68)와 다른 비율, 다른 방식(별이는 무겁게 / 빼콩은 발만 조심스럽게)

**보존 검증** (코드 구조로 보장 — 눈 분기가 존재하지 않는 경로):
| 항목 | 근거 |
|---|---|
| 하루 진행률 | `byeoli.worldX += byeoli.speed*dt` — 눈 계수 없음 |
| 소품 만남 빈도 | `byeoli.speed`는 `baseSpeed`에서만 설정, `updateEncounters`에 눈 분기 없음 |
| 행동 횟수·일기 생성량 | `maybeDiary`/`diaryFor`/`brain.decide` 무변경 |
| Authority·Threads·엽서·복구 | 렌더·타이머 외 미접촉, 게이트 12단 exit 0 |

**보류분 확장 지점만 마련** (값을 지어내지 않는다):
- `ENV.wind` — 렌더러의 옷자락 훅이 이 값만 읽는다. 소스가 없으므로 **항상 0 = 아무 움직임 없음**. 실제 바람은 이후 월드 환경 BUILD.
- `ENV.heat` / `ENV.coolingTags(['water','tree','eaves'])` — 향후 더위 상태와 coolingTarget 앞 쉼 가중치용 자리. **현재 어떤 행동도 발생시키지 않음** (그늘 판정 없이 행동 창작 금지).
