# MIMESIS Rule Engine

MIMESIS Sight Path는 3D 오브젝트를 예쁘게 배치하는 프로젝트가 아니다.

이 프로젝트의 핵심은 **사람이 자신의 기억을 걸어서 읽는 매체**를 만드는 것이다.

따라서 모든 생성 규칙은 장식보다 걷는 감각, 완성된 공간보다 희미해지는 기억, 오브젝트보다 시간을 우선한다.

## Core Principle

> 주인공은 카메라도 오브젝트도 UI도 아니다. 주인공은 걸어가는 시간이다.

## World Rules

### Rule 001 — 길은 끊기지 않는다

길은 불안정해도 된다.
무너져도 된다.
좁아져도 된다.
하지만 사용자가 걷고 있다고 느낄 수 있는 연속성은 유지해야 한다.

### Rule 002 — 완전한 오브젝트는 거의 등장하지 않는다

집은 집 전체보다 지붕, 창문, 벽 일부로 더 자주 등장한다.
방은 방 전체보다 책상, 의자, 바닥, 벽 한 면으로 더 자주 등장한다.
오브젝트는 현실의 물건이 아니라 기억의 남은 부분이어야 한다.

### Rule 003 — 오브젝트는 길보다 먼저 눈에 띄면 안 된다

길이 먼저다.
사용자는 오브젝트를 보러 가는 것이 아니라, 걷다가 기억의 흔적을 발견해야 한다.

### Rule 004 — 가까운 기억은 선명하고 먼 기억은 희미하다

거리감은 색, 투명도, 크기, 밀도, 형태의 완성도로 표현한다.
먼 곳의 오브젝트는 완전한 형태보다 실루엣이어야 한다.

### Rule 005 — 시대를 고정하지 않는다

현대도 아니고 과거도 아니고 중세도 아니고 미래도 아니다.
모든 시대가 조금씩 침식되어 섞인 곳이다.
마차, 시계탑, 안경, 아이소메트릭 방, 달, 소행성이 함께 있어도 된다.
단, 모두 같은 기억의 풍화 규칙 안에 있어야 한다.

### Rule 006 — 배치하지 말고 매몰시킨다

오브젝트는 길 위에 올려두기보다 길 아래, 절벽 단면, 먼 안개, 지층 사이에 일부만 드러나야 한다.

### Rule 007 — 방은 풍경보다 개인적인 기억이다

`isometric_rooms.fbx` 같은 방 자산은 매우 중요하다.
방은 집보다 더 개인적인 기억이다.
사용자는 풍경보다 방 하나, 책상 하나, 의자 하나에서 더 강하게 기억을 느낄 수 있다.

### Rule 008 — 안개는 화면을 덮지 않고 공간의 깊이가 되어야 한다

안개는 CSS 오버레이가 아니라 3D 공간의 거리감이어야 한다.
가까운 길은 보이고, 먼 기억은 희미해져야 한다.

### Rule 009 — 부품은 완성품보다 중요하다

집보다 지붕.
쉼터보다 기둥.
방보다 책상과 벽.
다리보다 끊어진 난간.
MIMESIS는 완성된 오브젝트의 전시장이 아니라 기억의 부품들이 침식된 공간이다.

### Rule 010 — 새로운 요소를 넣기 전에 묻는다

이것이 길을 더 걷고 싶게 만드는가?
이것이 기억을 더 깊게 만드는가?
이것이 오브젝트 자체보다 시간을 드러내는가?

셋 중 하나에도 답하지 못하면 넣지 않는다.

## Current Build Direction

BUILD 068 이후의 방향은 **Memory Parts**다.

완성된 모델을 그대로 배치하지 않고, 각 모델을 기억의 부품처럼 다룬다.

- House → roof / wall / window / floor
- Room → desk / chair / wall / floor
- Shelter → roof / pillar / bench
- Bridge → railing / broken span / stone base
- Clock → face / hand / tower fragment

이 부품들은 나중에 Asset Spawner가 기억의 농도와 거리, 길의 상태에 따라 자동으로 배치한다.
