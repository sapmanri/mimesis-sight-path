## 0. 합격 기준 (Public Beta 조건)

다음은 **전부 0건**이어야 한다:

- 치명적 JS 오류
- 데이터 유실
- 복구 실패로 인한 기록 유실
- Recovery Key 노출 (발급/복구 UI 외)
- 공유카드 전체 Observer Code 노출
- 모바일 UI 겹침
- 산책 중 강제 중단
- 재방문 시 온보딩 재노출
- 백업 충돌 후 무단 덮어쓰기

경고는 허용하되, **사용자가 계속 산책할 수 있어야 한다.**
**P0·P1 = 0건**이 Public Beta 조건. P2는 사용자 흐름을 막지 않는 것만 제한적으로 허용.

### 버그 등급
| 등급 | 정의 |
|---|---|
| **P0** | 데이터 유실 · 보안 노출 · 서비스 진입 불가 |
| **P1** | 복구 불가 · 주요 기능 실패 · 모바일 사용 불가 |
| **P2** | UI 겹침 · 오해되는 문구 · 반복 오류 |
| **P3** | 미세한 애니메이션·간격·표현 문제 |

## 2. 첫 진입 (온보딩)

- [ ] 첫 방문에 모달이 자동으로 뜬다
- [ ] 별이는 뒤에서 계속 움직인다
- [ ] 버튼에 포커스가 간다
- [ ] 모바일에서 화면 밖으로 넘치지 않는다
- [ ] 버튼을 누르면 자연스럽게 닫힌다
- [ ] 새로고침 후 다시 뜨지 않는다
- [ ] 기록 시트에서 "안내 다시 보기"가 된다
- [ ] 다시 보기 후에도 완료 플래그는 바뀌지 않는다

**관찰 포인트 (수정 금지):** 첫 사용자가 "기록 지키기"가 어디 있는지 헤매는가.
헤맨다고 바로 수정하지 말고, **몇 명에게서 반복되는지** 본다.

## 3. 산책 기본 동작

- [ ] 별이가 계속 이동한다
- [ ] 존 전환(마을→논길→숲→언덕)이 정상이다
- [ ] 시간대 변화가 정상이다
- [ ] 관찰일기가 계속 추가된다
- [ ] 긴 시간 켜도 프레임이 무너지지 않는다
- [ ] 탭을 다른 곳에 갔다 돌아와도 정상이다
- [ ] 화면 크기를 바꿔도 UI가 깨지지 않는다
- [ ] 세로·가로 회전 후 복구된다

## 4. 화면 유지 (Wake Lock)

- [ ] 첫 진입 시 Wake Lock 실패가 앱 전체 오류처럼 보이지 않는다
- [ ] 화면을 한 번 터치하면 재시도된다
- [ ] 허용 시 배지가 정상으로 바뀐다
- [ ] 탭 전환 후 돌아오면 다시 획득한다
- [ ] 지원하지 않는 브라우저에서도 산책은 계속된다

## 5. 기록 저장 (기록 지키기 + 자동 백업)

- [ ] "기록 지키기"가 눈에 띄는 위치에 있다
- [ ] Recovery Key는 발급 UI에서만 보인다
- [ ] 복사 버튼이 정확히 동작한다
- [ ] 발급 즉시 첫 백업이 성공한다
- [ ] 5분 자동 백업이 동작한다
- [ ] 탭 숨김(visibilitychange) 시 백업이 동작한다
- [ ] 자동 백업 실패가 산책을 막지 않는다
- [ ] 마지막 백업 상태가 조용히 표시된다 (배지)

## 6. 복구 (기존 별이 이어가기)

**필수 시나리오:**
- [ ] 다른 브라우저에서 정상 복구
- [ ] 다른 기기에서 정상 복구
- [ ] localStorage 삭제 후 복구
- [ ] 잘못된 Observer Code → "등록된 기록을 찾지 못했습니다"
- [ ] 잘못된 Recovery Key → "복구 코드가 일치하지 않습니다"
- [ ] 네트워크 끊김 → "연결하지 못했습니다" + 재시도
- [ ] 서버 오류 → "지금은 복구할 수 없습니다"
- [ ] 복구 도중 새로고침 → 부트에서 전진 완료/롤백, 반쪽 상태 없음
- [ ] 복구 도중 탭 종료 → 재진입 시 정상
- [ ] 로컬에 다른 기록이 있는 상태에서 복구 → 교체 확인 모달
- [ ] 보호 백업 실패 시 교체 중단 (강행 경로 없음)
- [ ] 충돌 발생 시 자동 백업 중지 + 조용한 배지
- [ ] 사용자 확인 없는 force 덮어쓰기 불가

**합격 기준: 복구 실패는 허용되지만, 기록 유실은 허용하지 않는다.**

## 7. 공유카드 PNG

- [ ] 카드 저장 성공
- [ ] 레이아웃 깨짐 없음
